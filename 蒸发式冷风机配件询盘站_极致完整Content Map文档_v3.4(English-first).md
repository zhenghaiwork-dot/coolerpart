# 蒸发式冷风机配件询盘站｜极致完整 Content Map 文档（English-first）v2

 **适用范围**：**B2B 优先**，客户可通过 **Google SEO + 站内自助筛选/兼容确认**提交询盘。设备范围：**柜式/箱体式（移动为主，兼有固定）**为主；**新款离心机式风系统**只做重点之一（但整体内容规模边缘化）；**不做风管式/工业通风式**。

 **核心销售方式（非常关键）**：
 - 用户多为“懂行采购/维修人员”，知道自己要的部件与关键参数。
 - 重点是让客户快速找到“对应类型 + 关键接口/尺寸/电参/性能”，并能做“差不多能装上/能替换”的自助选择。
 - **Model/Series（机型/系列）不是主匹配维度**：用于辅助证据（可选），不作为主要兼容入口。
 - **兼容/替换逻辑以接口与关键参数为主**。
 - **水泵和电机是高频核心部件**：导航与分类页承接要优先服务 Pump/Motor。

## 0. 核心目标（What & Why）

### 0.1 站点要解决的 3 个用户问题
1. **我需要的部件叫什么 / 属于哪一类**（配件/附件/零件/装配件/加固件/补充/增强/替换）

## 1. 语言与 SEO 命名规范（已确认：English 为主）

### 1.1 页面语言策略
- **SEO 元素（Title/H1/URL/Slug/Schema）以英文为主**。
- 页内规格表可做中英对照（可选），但核心可索引句子保持英文。

### 1.2 命名与拼写
- 统一使用通用英文品类名：`pump`、`motor`、`filter`、`cooling pad`、`valve`、`nozzle`、`gasket`、`mounting bracket`、`replacement parts`、`compatibility`。
- 避免把含糊词当作 SEO 主标题：`accessory` 只作为导航补充表达。

### 1.3 URL 结构总原则
- 结构清晰，包含层级与类型：
  - `/{system}/{area}/{product-type}/{seo-slug}`
- 例：
  - `/water-system/pumps/replacement-pump`
  - `/electrical-controls/motors/replacement-motor`
  - `/compatibility/interface/threaded-connection/`

## 5. 站内筛选字段（Faceted Search）与数据字典（Data Dictionary）

### 5.1 总原则（决定你们“能搜得准”的关键）
- 筛选默认按 **Interface-first** 与 **Parameter-first** 排序。
- **Model/Series 作为可选弱筛选或展示字段**，不作为主必填。

 适配你们的实际商品规模与客户类型：
- 不做 C 端那种“精细区间筛选”；**客户扫一眼就知道差不差，细节在 Request Quote 里确认**。

### 5.2 Facet 模式（建议）
- **Facet 1：Interface & Mounting**
  - **mounting_type（唯一用于筛选/列表标签/前端下拉筛选；枚举：B3 / B5 / B14 / Custom / Universal）**
  - Thread / Flange / Connector type
  - Bolt pattern / hole size / hole count（如有标准化数据）
  - 说明：**mounting_standard 仅用于详情页补充文本，不参与筛选**（系统根据 mounting_type 自动带出/填充默认标准文本，人工可微调）。

- **Facet 2：Dimensions (关键尺寸)**
  - pad size / frame size
  - inlet/outlet size
  - flange size
  - shaft diameter（电机轴相关）

- **Facet 3：Electrical / Performance（核心字段）**
  - Pump：**Lift Grade（pump_head_grade）**
  - Motor：**3 个独立下拉筛选**（Voltage / Power / RPM）；不做数值滑块（Frequency 不作为列表主筛选）
    - Voltage: 110V / 220V / 230V / 380V / 415V + Universal
    - Power(W): 250W / 370W / 550W / 750W / 1100W + Universal
    - RPM: 1300 / 1400 / 2600 / 2800 + Universal

**Electrical & Controls 分类页顶部筛选区（可直接照抄开发口径）**
- 位置：Electrical & Controls（电气电控）相关的分类/列表页顶部核心筛选区（列表页滚动首屏可见）。
- 形式：3 个下拉框垂直紧凑排列（竖向堆叠），从上到下依次为：**Voltage → Power(W) → RPM**。
- 交互：不做复合筛选、不做数值区间滑块；仅“单选下拉”。默认均为 **Any/All（不筛选）**。
- 枚举选项（固定不可变，避免选项膨胀）：
  - Voltage：110V / 220V / 230V / 380V / 415V / **Universal**
  - Power(W)：250W / 370W / 550W / 750W / 1100W / **Universal**
  - RPM：1300 / 1400 / 2600 / 2800 / **Universal**
- Universal 的含义：参数不完整/非标替换/缺少关键电参时的兜底归类；用于避免因数据缺失导致商品在筛选后被“筛空”。
- 与 SEO/索引一致性：下拉筛选会生成结果列表时该列表页统一 `noindex + canonical`；只允许系统页/子类目页/产品详情页/兼容专题页索引。

> Note: Motor mounting type (B3/B5/B14 etc.) is available in Structure & Mounting filter facets for further fitment matching.

提示：电机安装规格 mounting_type 不在本电气分类顶部筛选区，统一放置在「Structure & Mounting」结构安装类筛选组件内，用于二次装配适配筛选。

- **Facet 4：Material（可筛）**
  - pad material
  - pump seal material
  - housing material

- **Facet 5：System / Area**
  - Water/Evap/Air/Electrical…（用于SEO分类承接）

### 5.3 数据缺失策略
- 若某字段缺失：
  - 产品页不展示该字段（除“Pump Head/扬程 Lift Grade”统一枚举策略以外）
  - listing 的该 facet 不出现或标记为“unknown”（除“Pump Head/扬程 Lift Grade 统一枚举策略”以外）
- 不做“数字 + 文案混合”的复杂前端筛选，避免空结果与用户困惑。

### 5.4 Pump Head（扬程/提升）数据策略（方案 A：单字段枚举，少筛选）

 **目标**：前端只保留对用户一眼判断友好的少量筛选组件；具体参数差异在询盘时确认。

#### 5.4.1 统一字段与枚举档位
- **仅保留 1 个字段**：`pump_head_grade`
- 枚举值（下拉框选项）：
  - **Low Head**
  - **Medium Head**
  - **High Head**
  - **Extra Head**
  - **Universal**（完全缺失参考时）

#### 5.4.2 前端筛选逻辑（仅 1 个下拉框）
- Pump 列表页只提供 **Lift Grade 下拉筛选**
- 不再提供数值滑块/区间筛选（不做 2~3m 这类精细区间）
- 好处：避免 18% 无精确扬程 SKU 导致的空筛选结果；客户扫一眼即可判断是否“同档”。

#### 5.4.3 详情页文案兼容规则（两种情况）
- 有精准扬程（来自你们录入/原始资料时）：
  - 展示 `Max Head: {value} m`（同时可附 “(Medium Head)”）
- 无数值但有档位：
  - 仅展示 `Lift Grade: {grade text}`
- 若完全无任何参考：
  - 统一展示 `Lift Grade: Universal`

#### 5.4.4 扬程缺失时的联动辅助字段（提升“扫一眼”的判断效率）
在 Lift Grade 详情页仍建议同时突出：
- `power_w`（覆盖率高）
- `pump_flow_m3h`（与扬程适配强相关）
- `inlet_outlet_thread_size`（接口尺寸）
- `pump_type`（submersible / external centrifugal）

并在 Request Quote 前的快确认模块中提示：
- Lift Grade 匹配优先，其它参数用于二次确认与替代可行性判断。

## 6. 兼容/替代模块（Compatibility Engine）— 用证据链替代机型依赖

### 6.1 证据类型 Evidence type（建议字段）
每个兼容结论都要附带“证据类型”，便于懂行客户判断。

- **Evidence: Interface**（接口/安装方式一致）
- **Evidence: Dimensions**（关键尺寸一致/可适配）
- **Evidence: Electrical**（电参匹配/替代）
- **Evidence: Performance**（性能范围匹配）
- **Evidence: Cross-reference / Part number**（料号交叉引用：若有）
- **Evidence: Model/Series**（可保留，但不作为主匹配逻辑）

并且要有 **confidence** 标签：
- Guaranteed（有充分依据）
- Most common（高概率常用替换）
- Needs confirmation（需客户补图/补尺寸确认）
- May fit（可能适配但要注意差异点）

### 6.2 兼容页/产品页的“差异点说明”模板
- “What matches”：接口/尺寸/电参
- “What may differ”：可能不同的差异（例如：密封面、孔位、方向）
- “What you need”：需要的适配件/配套件（例如：gasket、coupling、adapter ring）

## 7. 询盘表单（RFQ Form）内容结构与优先级（配合 Pump/Motor）

### 7.1 表单分区（建议）
1. Contact info
2. Quote requirement（数量、MOQ、交期偏好、Incoterms 可选）
3. Compatibility inputs（兼容定位输入）
   - **优先（Pump/Motor 高频）**：
     - For Pump：inlet/outlet size、mounting type、**lift grade**（如有）
     - For Motor：voltage、power、RPM、shaft diameter、mounting_type
   - **二次确认（可选）**：
     - Photo upload（部件图片）
     - Dimensions（尺寸关键字段）
     - Interface description（安装方式描述）
   - **Model/Series（弱可选）**：可作为辅助输入。
4. Technical notes（故障症状，如 leak/clog/pump failure/motor issue）
5. 提交

### 12.2 关系（Relationships）
- ProductType 属于 Area 与 System
- Product 属于 ProductType
- InterfaceSpec / DimensionSpec / ElectricalSpec / PerformanceSpec 属于 Product
- CompatibilityEvidence 将 Product 连接到“兼容结论”与“hub页面索引键”
- RelatedProductsRule 生成详情页的 Related Products 区块
