# 2026-07-10

## 站点阅读与架构分析

对 CoolerParts 蒸发式冷风机配件询盘站进行了全面阅读和分析。

### 项目概要
- **技术栈**: Astro 7 (SSG) + Tailwind CSS v4 + TypeScript，基于 AstroWind 模板
- **定位**: B2B 蒸发式冷风机配件询盘平台（非电商，无支付结算）
- **核心理念**: Interface-first matching（接口优先匹配），弱化 Model/Series 匹配
- **语言**: English-first SEO，i18n = en

### 页面架构（4层）
1. **Homepage** (`/`) — Hero + 7 系统分类卡片 + "How It Works" 三步流程
2. **System pages** (`/[system]/`) — 7 个系统页，展示子分类 + 前4个产品预览
3. **Area pages** (`/[system]/[area]/`) — 29 个子分类页，产品网格列表
4. **Product detail** (`/products/[slug]/`) — 12 个产品详情页，含规格/兼容性/FAQ/询盘按钮

### 支撑页面
- **Compatibility guide** (`/compatibility/`) — 4种匹配方式入口（接口/尺寸/电气/交叉引用），目前为静态展示
- **Inquiry/RFQ** (`/inquiry/`) — 询盘表单 + 侧边栏询盘篮
- **About / Contact** — 企业信息与联系方式

### 数据层
- `src/data/systems.ts` — 7 个系统 + 29 个子分类的结构定义
- `src/data/products.ts` — 12 个产品数据（水泵×2, 密封×1, 滤网×1, 电磁阀×1, 湿帘×2, 电机×2, 电容×1, 支架×1, 滤芯×1, 脚轮×1）
- `src/data/inquiry-types.ts` — TypeScript 类型定义
- `src/scripts/inquiry-basket.js` — 浏览器端询盘篮逻辑（基于 localStorage + CustomEvent）

### 询盘流程
1. 用户浏览产品 → 点击 "Add to Inquiry List" 或 "Request Quote"
2. 询盘篮数据通过 browser storage 持久化，跨页面同步
3. 进入 `/inquiry/` 填写表单，侧边栏自动回填产品信息
4. 提交到 Formspree（占位 action，需配置真实 ID）

### Content Map 文档
- 存在详尽的 v3.4 English-first Content Map 文档，定义了完整的类目树、筛选字段（Faceted Search）、兼容性引擎（Evidence + Confidence 标签）、RFQ 表单结构、SEO 模板等
- 当前代码实现是 Content Map 的第一阶段（基础架构 + 数据 + 页面骨架），尚未实现筛选器、兼容性引擎、RFQ helper pages 等高级功能
